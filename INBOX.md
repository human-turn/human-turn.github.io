# Inbox — сырые идеи материалов

Лови идею одной строкой, как пришла. Без структуры, без оформления.
Раз в неделю разгребаем: идея → черновик (`draft: true`) в нужном разделе → строка в `REVIEW.md`. Что не взлетает — вычёркиваем.

Конвейер: **инбокс → черновик → на правках → принято**. Сайт первичен.

## Идеи

- [ ] LiteLLM как эгресс-контроллер — gateway для исходящих LLM-вызовов: маршрутизация по разрешённым провайдерам, лимиты/бюджеты, контроль в росреалиях
- [ ] Дашборд Claude agents — что это и зачем (наблюдение за параллельными агентами)
- [ ] Git worktree — что это и зачем; удобно, когда много агентов работают в одном проекте параллельно. Добавить ссылку на дашборд агентов
- [ ] Симлинки в `.bashrc` / `.zshrc` — подключать локальные модели для приватных данных + про переменную с токеном
- [x] Команда `fork` — для «чистоты» контекста → `concepts/fork-context.md` (черновик); `/branch` — заготовка `concepts/branch-sessions.md`
- [ ] MCP IDEA как gate — JetBrains IDEA в роли управляемого шлюза для доступа к ресурсам (например, к БД)

### Claude Code / агенты

- [ ] Hooks (PreToolUse/PostToolUse/Stop и пр.) — событийная автоматизация: блокировка опасных команд, авто-формат после правок, gate-проверки
- [ ] Субагенты и оркестрация — изоляция контекста через дочерних агентов; когда делегировать, когда оверхед не оправдан; продолжение /fork и /branch
- [ ] Plan mode — режим планирования без правок: агент строит план → ты утверждаешь → исполнение
- [ ] Checkpoints / rewind — откат состояния кода в сессии
- [ ] Headless-режим и Agent SDK — `claude -p` в скриптах/CI; с 15.06.2026 SDK/headless/GH Actions уходят на отдельные API-кредиты вместо подписки
- [ ] Claude Code в GitHub Actions — агент как шаг пайплайна по @-меншену в issue/PR
- [ ] Plugins-экосистема и эволюция скиллов — `claude plugin init`, авто-загрузка `.claude/skills`, `/reload-skills`, `reloadSkills` в SessionStart-хуках; дополняет страницу про каталоги скиллов

### AI-инфра и модели

- [ ] Prompt caching на практике — кэш контекста между вызовами (−90% стоимости / −85% латентности на длинных промптах); как структурировать промпт под кэш
- [ ] Batch API — асинхронная пакетная обработка со скидкой ~50% для офлайн-задач (эвалы, массовая разметка)
- [ ] LLM-observability: Langfuse vs Helicone — трейсинг промптов, атрибуция стоимости, eval-скоры; LiteLLM про роутинг, эти про наблюдаемость
- [ ] vLLM vs Ollama для self-host — Ollama для dev/прототипа, vLLM для прод-нагрузки (бенчмарк 793 vs 41 TPS); дополняет страницы про локальные модели
- [ ] RAG-инфра по стадиям — chunking → embeddings → vector store → reranking → semantic cache → eval, с цифрами recall@k и стоимости

### Процессы команды

- [ ] Spec-driven development — спека как источник правды для агента вместо vibe-coding (GitHub Spec Kit, AWS Kiro); дополняет OpenAPI-контракты
- [ ] Evals для LLM-фич — тестирование недетерминированных фич: curated question set, LLM-as-judge, оффлайн eval-петля в CI
- [ ] AGENTS.md vs CLAUDE.md — складывающийся стандарт памяти для агентов: разница, паттерны, кросс-инструментальность
- [ ] Continuous background code review для агентов — фоновое ревью каждого коммита в петлю агента; иной угол, чем AI-ревью в MR
- [ ] Контекст-инженерия в больших кодбазах — что подавать агенту, чем грузить субагентов, компакция
