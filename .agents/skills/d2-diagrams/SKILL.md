# D2-диаграммы HumanTurn

## Главное правило

**Цвета не задаём вручную.** Палитра «Гжель» зашита в `src/styles/d2-gzhel-theme.d2` через `theme-overrides`. Remark-плагин `src/plugins/remark-d2-theme.js` автоматически вставляет `vars`-блок в каждый ````d2`-код перед сборкой. Единственный источник цветов — `src/styles/gzhel.css`.

## Workflow

1. **Справка по синтаксису.** `query-docs` с `libraryId: /terrastruct/d2-docs` по нужной фиче.
2. **Пиши `.d2`.** Следуй правилам ниже. Тема применится автоматически.
3. **Не рендери.** `astro-d2` собирает SVG при старте dev-сервера (`npm run dev:force`).

> Если диаграмме нужна **своя** тема — добавь `vars:`-блок вручную. Плагин пропустит блоки, где `vars:` уже есть.

## Тема «Гжель»

Автоматически применяется ко всем диаграммам. D2 сам распределяет токены по ролям:

| Токен | Роль                                | Цвет                          |
| ----- | ----------------------------------- | ----------------------------- |
| B1    | обводки контейнеров                 | `#16306f` (accent-high)       |
| B2    | обводки фигур                       | `#1c2a52` (gray-1)            |
| B3    | основные фигуры                     | `#2f54b0` (accent)            |
| B4    | заливка контейнеров верхнего уровня | `#8593b5` (gray-4)            |
| B5    | заливка вложенных контейнеров       | `#d3e1f9` (accent-low)        |
| B6    | заливка глубоких фигур              | `#e8effc`                     |
| N1–N7 | текст → фон                         | сине-серая шкала из gzhel.css |

## Структура диаграммы

### Харнесс — протагонист

Харнесс — герой сайта. На **любой** схеме, где он есть, выделяем его одинаково, чтобы взгляд цеплялся за один якорь across диаграмм:

```d2
agent: Харнесс {
  style.fill: "#d3e1f9"
  style.stroke: "#2f54b0"
  style.stroke-width: 3
}
```

### Контейнеры

Группируем логические блоки. Направление внутри контейнера можно переопределить.
**Заголовки скрываем только у технических контейнеров** — обёрток для layout'а и сиквенс-диаграмм.

```d2
direction: down

network: Внутренняя сеть {
  direction: right
  workstation: Рабочая станция {
    agent: Харнесс
  }
}

# Сиквенс-диаграмма без заголовка
flow: "" {
  shape: sequence_diagram
  ...
}
```

### Сиквенс-диаграммы

Для пошаговых сценариев используем `shape: sequence_diagram`. Логические этапы группируем в **фазы** (labeled groups) — они обводятся рамкой с заголовком:

```d2
flow: "" {
  shape: sequence_diagram

  llm: LLM-провайдер
  harness: Харнесс
  mcp: MCP-сервер

  discovery: Знакомство {
    style.fill: "#eef2fb"      # фаза кобальтового тона
    style.font-color: "#16306f"
    style.bold: true
    harness -> mcp: Инициализация {
      style.stroke: "#2f54b0"
      style.font-color: "#1c2a52"  # тёмный текст сообщения — контраст на фоне полосы
    }
  }

  call: Вызов {
    style.fill: "#e8f2f2"      # вторая фаза — бирюзовый тон, чтобы фазы различались не только разрывом
    style.font-color: "#1d5a64"
    style.bold: true
    llm -> harness: Вызови инструмент {
      style.stroke: "#2d7d8a"
      style.font-color: "#1c2a52"
    }
  }
}
```

**Правила сиквенса:**

- **Текст сообщений** — всегда `style.font-color: "#1c2a52"`. По умолчанию подписи бледно-серые и тонут на фоне полосы фазы.
- **Фазы тонируем в разный оттенок** (`style.fill`) под цвет их стрелок: кобальт `#eef2fb`, бирюза `#e8f2f2`. Заголовок фазы — `bold` + цвет в тон.

### Формы по смыслу

- База данных → `shape: cylinder`
- Очередь/шина → `shape: queue`
- Пользователь/роль → `shape: person`
- Документ → `shape: page`
- Облако → `shape: cloud`
- Остальное → `shape: rectangle` (по умолчанию)

Не выдумывать значения `shape` — сверяться с Context7.

### Именование

- Ключи нод — латиница, нижний регистр, без пробелов: `agent`, `corp_api`
- Лейблы — человеческий язык: `Харнесс`, `Корпоративные системы`

### MCP-серверы как субпроцессы

MCP-сервер — **отдельный субпроцесс**, порождённый харнессом, а не часть его. Рисуем не внутри бокса Харнесса, а группой-обёрткой рядом, на той же рабочей станции. Цепочка: Харнесс → MCP-сервер → целевая система.

```d2
mcp: MCP-серверы · субпроцессы {
  jira_mcp: jira-mcp
}
# Харнесс ↔ сервер — по протоколу MCP (бирюза, пунктир)
agent <-> mcp.jira_mcp: MCP { style: { stroke: "#2d7d8a"; stroke-dash: 4 } }
# Сервер ↔ система — «родным» транспортом (сланец, сплошная): HTTP, CDP, ...
mcp.jira_mcp <-> jira: HTTP { style: { stroke: "#50608a" } }
```

Прямой доступ (файлы/диск) идёт **без** MCP — сплошная по умолчанию.

## Соединения и стрелки

### Типы стрелок

| Тип              | Синтаксис | Когда                         |
| ---------------- | --------- | ----------------------------- |
| Однонаправленная | `a -> b`  | Прямой доступ, вызов          |
| Двусторонняя     | `a <-> b` | Протокольный обмен (MCP, LLM) |
| Без направления  | `a -- b`  | Ассоциация                    |

Двусторонние `<->` предпочтительнее двух отдельных встречных — чище.

### Анимации

**Не используем.** `animated: true` не применяем нигде — ни на архитектурных схемах, ни на сиквенсах.

### Стилизация стрелок

**Обычная связь** — без стилей, идёт темой по умолчанию:

```d2
agent -> disk
```

**Протокольная (MCP)** — пунктир, бирюзовый:

```d2
agent <-> server: {
  style: {
    stroke: "#2d7d8a"
    stroke-dash: 4
  }
}
```

**Внешний LLM-провайдер** — кобальт, пунктир:

```d2
agent <-> llm: {
  style: {
    stroke: "#2f54b0"
    stroke-width: 2
    stroke-dash: 4
  }
}
```

### Цвета стрелок

Единственное место, где цвет задаётся явно (не из темы):

| Назначение                      | Цвет                         |
| ------------------------------- | ---------------------------- |
| MCP-серверы                     | `#2d7d8a` (бирюзовый)        |
| Родной транспорт сервер→система | `#50608a` (сланец, сплошная) |
| LLM-провайдер                   | `#2f54b0` (кобальт)          |
| Фазы в сиквенсе                 | разные цвета для разных фаз  |
| Предупреждение/риск             | `#e57373` (мягкий красный)   |
| Успех/подтверждение             | `#66bb6a` (зелёный)          |

### Позиционирование

Принудительное размещение — невидимая связь:

```d2
llm -> network: { style: { opacity: 0 } }
```

Но лучше полагаться на естественный layout: `direction` + порядок объявления нод.

## Диаграммы в табах

````mdx
import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs>
	<TabItem label="Потоки данных">```d2 ... ```</TabItem>
	<TabItem label="Компоненты и связи">```d2 ... ```</TabItem>
</Tabs>
````

## Границы

- Только `.d2` код. Не запускаем `d2` CLI — рендер через `astro-d2`.
- Не добавляем `vars`-блок — плагин `remark-d2-theme` делает это автоматически.
- Не выдумываем цвета — либо из темы, либо из таблицы «Цвета стрелок».
- Не выдумываем `shape`-значения — сверяемся с Context7.
- Не используем `animated: true`.
