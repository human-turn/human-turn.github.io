# AGENTS.md

Документационный сайт **HumanTurn** на Astro + Starlight: знания об AI-разработке — агенты, скиллы, инфраструктура инференса, практика внедрения.

> **Дух проекта:** удобная среда AI-разработки. Мы копим проверенный опыт (pattern language), растим связанный граф материалов, учитываем российскую специфику (росреалии). Сайт первичен — книга собирается позже срезом.

## Команды

```bash
npm run dev          # dev-сервер на http://localhost:4321 (strictPort)
npm run dev:force    # сброс content layer cache (вместо rm -rf .astro)
npm run build        # прод-сборка в ./dist
npm run preview      # локальный просмотр собранного
npm run typecheck    # astro check
npm run lint         # eslint .
npm run lint:fix     # eslint . --fix
npm run format       # prettier --write .
npm run format:check # prettier --check .
```

Перед коммитом: `npm run format && npm run lint && npm run typecheck` — все три зелёные.

**Порт 4321, `strictPort`.** Если «port in use» — сервер уже запущен, новый не поднимай.

**Залипла картинка/frontmatter в dev?** HMR кэширует ассеты. `npm run dev:force` (= `astro dev --force`).

**GitHub:** утилита `gh` доступна.

## Структура контента

Страницы в `src/content/docs/`. **Slug = путь файла** (без расширения), не прописывается во frontmatter. Навигация — гибридный сайдбар в `astro.config.mjs`.

Актуальная структура папок: [`src/content/docs/AGENTS.md`](src/content/docs/AGENTS.md).

### Гибридный сайдбар

Группы верхнего уровня — вручную в `astro.config.mjs`. Внутри групп — `autogenerate`: новый файл в папке сам появляется в сайдбаре.

**Порядок:** через плагин `starlight-auto-sidebar`:

- Папки: `_meta.yml` с полем `order: N`
- Страницы: `sidebar: { order: N }` во frontmatter

**Полная инструкция:** скилл `sidebar-management`.

### Подводные камни Starlight

- **Starlight ≥ 0.39**: короткая форма `{ label, autogenerate }` удалена. Группу обязательно оборачивать:
  ```js
  { label: 'Руководства', items: [{ autogenerate: { directory: 'guides' } }] }
  ```
- Любой `slug` в сайдбаре должен соответствовать существующей странице, иначе сборка падает.
- **llms.txt:** при изменении структуры разделов — синхронизировать `customSets` в `@rttnd/starlight-llms-txt` (в `astro.config.mjs`), иначе `llms.txt` и `_llms-txt/*.txt` рассинхрон.

### Конвенция frontmatter

```yaml
---
title: '...'
description: '...' # желательно
draft: true # пока не готово к публикации
tags: [инструменты, how-to] # обязательно: домен + тип
---
```

**Slug не прописывается** — он равен пути файла. Index-страницы разделов: slug = имя папки.

**Словарь тегов:** [`TAGS.md`](TAGS.md) — TODO: пополнять при добавлении статей.

### Статусы и публикация

`draft: true` — страница видна в `npm run dev`, но **не в прод-сборке**. Готово к публикации → `draft: false`.

### Терминология

**Все статьи используют единую терминологию.** Канонический справочник: [`src/content/docs/library/glossary.md`](src/content/docs/library/glossary.md).

- Нельзя в одной статье писать «харнесс», в другой «агент», в третьей «хост».
- Каждый термин имеет **один** русский вариант и список запрещённых синонимов.
- При добавлении статьи — сверять термины со справочником.
- При ревью статьи — проверять гомогенность терминологии.
- Новый термин → сначала добавляется в справочник, затем используется в статьях.

### Связанный граф и паттерны

- **Граф, а не дерево.** У каждой страницы блок `## Связанные материалы` со ссылками на соседние.
- **Практики и концепты** — по каркасу (pattern format по Крису Ричардсону): **Контекст → Проблема → Предпосылки → Решение → Последствия → Связанные материалы**. Безболезненных решений не бывает — в последствиях обязательны минусы и новые проблемы.
- **Туториалы и how-to** — свободная форма, но `## Связанные материалы` обязателен.
- **Росреалии** — импортозамещение моделей, 152-ФЗ, доступность API — тег `росреалии`, inline в контексте или последствиях, отдельным блоком только если тема посвящена.
- **Идеи** копим в `INBOX.md`: инбокс → черновик → публикация.

### Добавить страницу

1. Идея → `/idea` (пишет строку в `INBOX.md`).
2. Когда дозрела → `/writing-article` (ведёт через интервью, создаёт черновик).
3. Или создать `.md`/`.mdx` в нужной папке напрямую, frontmatter по конвенции, tags, `draft: true`.
   - **`.mdx`** — если страница использует компоненты (`<Tabs>`, `<TabItem>`, `import`).
   - **`.md`** — для обычных страниц без компонентов.
4. `autogenerate` подхватит сам. Для порядка — `sidebar: { order: N }` во frontmatter.

## Качество кода

- **TypeScript**: `astro/tsconfigs/strictest`, без `any`, без `as` без необходимости.
- **ESLint**: flat config, `typescript-eslint` + `eslint-plugin-astro` + `eslint-config-prettier`.
- **Prettier**: табы, одинарные кавычки, `printWidth: 100`, `proseWrap: preserve`, `prettier-plugin-astro`.
- **Алиас**: `~/*` → `src/*`.
- **a11y-линтинг отложен** (eslint-plugin-jsx-a11y ждёт ESLint 10).

- Работаешь с сайдбаром/структурой → `sidebar-management`
