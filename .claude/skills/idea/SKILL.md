# Захват идеи в INBOX.md

Цель — нулевое трение: автор бросает мысль на ходу (часто голосом), скилл доводит её до аккуратной строки в правильном месте. Уточняющие вопросы убивают сценарий — не задавай их; если чего-то не хватает, просто фиксируй как есть.

## Шаги

1. Прочитай `INBOX.md` в корне репо.
2. Сожми мысль в одну строку в формате существующих записей:

   ```
   - [ ] Короткое название — суть одним предложением; чем дополняет существующие страницы (если очевидно)
   ```

   - Сохраняй формулировки автора, убирай только «воду». Не добавляй фактов, которых автор не говорил, — строка должна остаться _его_ мыслью.
   - Если автор явно сказал, что уже **внедрил** на предприятии или **пробовал** сам — добавь в конец метку `[внедрено]` или `[пробовал]`. Если это «надо бы изучить» — `[теория]`. Нет сигнала — нет метки, не угадывай.

3. Выбери секцию по теме:
   - `### Claude Code / агенты` — возможности Claude Code, субагенты, хуки, плагины, SDK;
   - `### AI-инфра и модели` — провайдеры, прокси, self-host, кэширование, observability;
   - `### Процессы команды` — ревью, контракты, спеки, организация работы, обучение;
   - `### Росреалии` — импортозамещение, 152-ФЗ, доступ к API из РФ;
   - `## Идеи` (общий список сверху) — всё, что не легло в секции. Новые секции не создавай — секции пересматриваются отдельно, при разгребании.
4. Добавь строку в конец выбранной секции. Чужие строки не редактируй.
5. Если мысль дублирует существующую строку — дубль не добавляй: покажи автору существующую запись и дополни её, только если новая формулировка добавляет смысл.

## Ответ автору

Одна-две строки: секция + добавленная строка. Не коммить и не предлагай коммит — инбокс коммитится пачкой при еженедельном разгребании.
