## Зачем

План-мод агента вываливает простыню markdown в терминал. Ты скроллишь, читаешь, пытаешься не потерять мысль. Чтобы ответить — пишешь в текстовое поле пронумерованный список «исправь пункт 3, добавь в пункт 7, пункт 12 не нужен». Когда пунктов под 20, ориентироваться невозможно.

Встроенные ask-тулы агентов задают вопросы, но не дают комментировать **произвольные куски текста** прямо по месту.

## Решение

[Plannotator](https://plannotator.ai/) — open-source веб-интерфейс для ревью планов и кода AI-агентов с контекстным аннотированием. Комментарии привязаны к конкретным местам плана или диффа — как в Google Docs. Аннотации возвращаются агенту структурированным фидбеком, при необходимости — в несколько проходов.

Поддерживает **Pi** и **Claude Code**. Работает локально — никаких внешних API.

## Что меняется

| Было                       | Стало                                                                |
| -------------------------- | -------------------------------------------------------------------- |
| Стена текста в терминале   | План в браузере с рендерингом markdown                               |
| Нумерованный список правок | Контекстные аннотации на конкретных фрагментах                       |
| Без диаграмм               | Mermaid-диаграммы рендерятся графически                              |
| Один проход                | Итерации: фидбек → агент переделывает → Plan Diff показывает разницу |

## Как работает

Plannotator встраивается в стандартную парадигму агента — не требует отдельной команды для план-ревью.

1. Агент формирует план и сохраняет в `PLAN.md`
2. Plannotator открывает его в браузере:
   - План отображается как отрендеренный markdown
   - Mermaid-диаграммы рендерятся графически
   - Выделяешь текст → панель аннотаций: **удалить**, **комментарий**, **быстрый лейбл** (⚡), **«выглядит хорошо»** (👍)
3. Читаешь план и размечаешь — UI спроектирован для скоростной работы: хоткеи на все действия, разметка без отрыва от чтения
4. **Approve** — агент начинает реализацию; **Send Feedback** — аннотации уходят агенту: что удалить, что изменить, что добавить
5. Агент переделывает план и снова отображает его. **Plan Diff** показывает, что именно изменилось (добавлено/удалено) — не надо перечитывать всё заново

## Команды

### `/plannotator`

План-ревью. Основной сценарий. В Pi можно запустить режим планирования командой или `Ctrl+Alt+P` — агенту останется доступ на запись только к `PLAN.md`, остальное ограничено. Когда итерации планирования завершаются, план открывается в браузере для обсуждения.

### `/plannotator-review`

Ревью кода. Классический git diff-вьювер для ревью изменений. Позволяет комментировать строки и файлы — интерфейс аннотаций такой же, как в плане.

### `/plannotator-last`

Последнее сообщение. Открывает последнее агента в визуальном редакторе. Удобно, когда агент выдал длинный ответ и хочется обсудить конкретные места путём их локального комментирования, а не отвечать общим текстом.

### `/plannotator-annotate`

Аннотирование файлов. Открывает произвольный markdown-файл в редакторе аннотаций. Полезно для правок документации, ревью спек, написания статей — например, эта статья правится через Plannotator.

Поддерживает локальные `.md`, `.html`, а также URL (страница скачивается и конвертируется в markdown).

## Установка

<Tabs>
<TabItem label="Claude Code">

```bash
curl -fsSL https://plannotator.ai/install.sh | bash
```

Ставит утилиту `plannotator` и слэш-команды в `~/.claude/skills/`. Затем в самом Claude Code — плагин для хука plan mode:

```
/plugin marketplace add backnotprop/plannotator
/plugin install plannotator@plannotator
```

Перезапусти Claude Code. Первая команда даёт CLI + `/plannotator-review`, `/plannotator-annotate`, `/plannotator-last`; вторая — хук на `ExitPlanMode`.

</TabItem>
<TabItem label="Pi">

Одна команда — расширение ставится через npm:

```bash
pi install npm:@plannotator/pi-extension
```

Команда `plannotator` и хоткей доступны сразу. План-мод включается через `--plan` или слэш-команду `/plannotator`.

</TabItem>
</Tabs>

## Известные проблемы

- **Маркетплейс-плагин ≠ полная установка (Claude Code).** Плагин из маркетплейса ставит только `PermissionRequest` хук для plan mode. Слэш-команды `/plannotator-review`, `/plannotator-annotate`, `/plannotator-last` и бинарь `plannotator` в них не входят — их ставит `curl -fsSL https://plannotator.ai/install.sh | bash`. Если поставить только плагин, команд не будет, а хук plan mode молча упадёт (нет бинаря в PATH).

- **Approve с аннотациями в Claude Code.** Claude Code не поддерживает передачу фидбека вместе с approve. Если хочешь оставить пометки — используй Send Feedback. В Pi такой проблемы нет.

- **После plan mode — сразу реализация.** Plannotator не предлагает начать новый чат — агент выходит из план-мода и пишет код в ту же сессию. Контекст плана и реализации смешивается.
