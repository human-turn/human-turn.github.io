Когда агент запускает `npm run dev`, он залипает: процесс висит, агент ждёт,
контекст простаивает. **pi-tmux** решает это — даёт агенту инструмент `tmux`
для запуска долгих команд в отдельных панелях.

## Установка

```bash
pi install npm:@ogulcancelik/pi-tmux
```

Или разовый запуск без установки:

```bash
pi -e npm:@ogulcancelik/pi-tmux
```

**Требования:** tmux, pi ≥ v0.40, pi запущен внутри tmux-сессии.

## Что даёт

Пять действий через инструмент `tmux`:

| Действие | Что делает                                  |
| -------- | ------------------------------------------- |
| `run`    | Запускает команду в именованной панели      |
| `read`   | Захватывает вывод из панели                 |
| `send`   | Посылает клавиши (`C-c`, `Enter`) или текст |
| `stop`   | Убивает именованную панель                  |
| `list`   | Список управляемых панелей                  |

Агент сам решает когда и как использовать: `bash` для коротких команд,
`tmux run` для долгих.

## Примеры

### Запуск dev-сервера

Агент может запустить dev-сервер и продолжить работу:

```text
tmux run name="dev" command="npm run dev"
```

Сервер крутится в соседней панели, агент не заблокирован.

### Проверка вывода

```text
tmux read name="dev"
```

Захватывает последний вывод панели и отдаёт агенту.

### Остановка

```text
tmux send name="dev" keys="C-c"
tmux stop name="dev"
```

## Как это работает

- Панели тегаются через `@pi_name` — надёжный discovery
- ANSI-escape последовательности вырезаются из захваченного вывода
- Мёртвые панели автоматически заменяются при `run`
- Расширение самоотключается если pi не в tmux (проверка `$TMUX`)
- Также самоотключается при наличии `HERDR_ENV` (уступает herdr)

## Ограничения

- **Только внутри tmux.** Без tmux-сессии инструмент молча не регистрируется.
- **Нет нотификаций.** Агент не узнает что команда завершилась — надо явно `read`.
- **Не для интерактивных CLI.** Для редакторов/REPL — смотри `pi-interactive-shell`.
