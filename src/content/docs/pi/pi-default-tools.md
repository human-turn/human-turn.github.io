По умолчанию Pi даёт модели четыре инструмента: `read`, `write`, `edit`, `bash`.
Ещё три — `grep`, `find`, `ls` — есть в поставке, но неактивны. Их можно включить
флагом `--tools` при запуске, но это неудобно: каждый раз помнить и прописывать.

Пакет **pi-default-tools** решает проблему: список инструментов прописывается
в `settings.json` и подхватывается при старте любой сессии.

## Установка

```bash
pi install npm:@johnnywu/pi-default-tools
```

## Настройка

Добавь ключ `tools` в `~/.pi/agent/settings.json` (глобально) или
в `.pi/settings.json` (на уровне проекта):

```json
{
	"tools": ["...", "grep", "find", "ls"]
}
```

Маркер `"..."` означает «сохрани текущие дефолтные инструменты». Без него
массив работает как строгий whitelist — например, `["grep", "find", "ls"]`
оставит только эти три и уберёт `read`, `write`, `edit`, `bash`.

## Как это работает

| Конфигурация             | Эффект                                                  |
| ------------------------ | ------------------------------------------------------- |
| `tools` отсутствует      | Дефолтные инструменты (`read`, `write`, `edit`, `bash`) |
| `tools: ["...", "grep"]` | Дефолтные + `grep`                                      |
| `tools: ["grep"]`        | **Только** `grep`, остальные отключены                  |

### Приоритет

Настройки проекта (`.pi/settings.json`) переопределяют глобальные
(`~/.pi/agent/settings.json`). Если в проекте ключ `tools` не задан —
используется глобальный.

## Зачем это нужно

- **grep** — поиск по кодовой базе без вызова `bash`, модель использует
  эффективнее и структурнее
- **find** — поиск файлов по имени/маске
- **ls** — листинг директорий, тоже напрямую, без `bash`

С этими инструментами агент меньше дёргает shell и быстрее ориентируется
в проекте. Особенно заметно на больших кодбазах, где `bash`-вариант (`grep -r`,
`find .`) возвращает тонны вывода и быстрее забивает контекст.
