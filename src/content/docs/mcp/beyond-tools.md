## Введение: MCP ≠ tool calling

- Первое впечатление: MCP = способ вызвать тул у агента
- На самом деле протокол двусторонний: сервер может запрашивать у клиента действия во время исполнения
- Это радикально меняет паттерны интеграции
- Что рассмотрим: sampling, elicitation, notifications/logging, roots, resources, prompts, tasks

## Двусторонняя связь: сервер говорит с клиентом

- В классическом RPC: клиент вызвал → сервер ответил. Всё.
- В MCP: сервер во время обработки может отправить клиенту:
  - Прогресс (progress notification) — «20% готово»
  - Лог (logging) — «обработано 1000 строк»
  - Запрос к LLM (sampling) — «сгенерируй описание для этого блока»
  - Запрос к пользователю (elicitation) — «подтверди удаление»
  - Ограничения ФС (roots) — «в каких директориях мне можно работать?»
- Клиент должен быть готов обработать эти reverse-запросы — capability negotiation при handshake

## Sampling: сервер просит LLM

- Сервер во время выполнения тула говорит клиенту: «отправь это в LLM и верни ответ»
- Сценарии:
  - Сервер агрегирует данные, LLM их интерпретирует
  - Сервер достал сырые логи → LLM пишет саммари
  - Сервер нашёл 50 кандидатов → LLM выбирает топ-5
- Не все клиенты поддерживают: Claude Code — да, некоторые IDE — нет

## Elicitation: сервер спрашивает пользователя

- Сервер прерывает выполнение и запрашивает ввод: подтверждение, выбор из вариантов, дополнительный параметр
- Сценарии:
  - «Точно удалить таблицу? [Y/n]»
  - «Нашёл 3 окружения: dev / staging / prod. Куда деплоить?»
- Отличается от sampling: elicitation идёт к человеку, а не к LLM

## Notifications и логирование

- Сервер шлёт асинхронные уведомления: прогресс, предупреждения, статус
- Клиент может подписаться на уровни логов (debug, info, error)
- Полезно для длительных операций: миграции, экспорт данных, бэкап

## Roots: границы файловой системы

- Сервер спрашивает клиента: «к каким директориям у меня есть доступ?»
- Клиент отвечает списком разрешённых корней (project root, home, ничего)
- Это механизм безопасности: сервер не шарит по всей ФС без спроса

## Resources и Prompts

- **Resources** — сервер предоставляет данные, которые агент может читать (файлы, строки БД, API-ответы). Не тулы — пассивные данные.
- **Prompts** — сервер предоставляет шаблоны промптов, которые агент может использовать. «Проведи код-ревью этого файла», «Сгенерируй changelog».
- Вместе с Tools образуют полный набор: тулы = действия, ресурсы = данные, промпты = инструкции

## Tasks (MCP 2026-07-28)

- Долгоиграющие операции: агент запустил — сервер выполняет асинхронно — агент забрал результат позже
- Не блокирует сессию, не требует постоянного соединения
- Поверх Streamable HTTP, вписывается в stateless-ядро новой версии протокола
