Lottie — это анимация, описанная обычным JSON: слои, ключевые кадры, кривые. Раз
это текст, его умеет генерировать языковая модель. Значит, чтобы получить
анимацию, **не нужны ни After Effects, ни облачный редактор, ни аккаунт** —
достаточно Claude Code, который пишет `.json` прямо в проект.

Этот гайд — про основной, полностью локальный путь. В конце — пара альтернатив на
случай, когда нужен терминальный CLI или визуальный редактор.

## Подготовка окружения

Голая модель пишет Lottie «механически»: тайминг плоский, структура местами
невалидная. Два скила убирают эту проблему — оба ставятся локально и работают без
аккаунта.

```bash title="Ставим скилы один раз"
# Принципы движения: тайминг, easing, хореография
npx skills add LottieFiles/motion-design-skill

# Предметный скил под Lottie: SVG → валидный Lottie JSON
npx skills add obeskay/lottie-animator-skill
```

- **`motion-design-skill`** даёт «режиссёрское» мышление — какую кривую взять для
  отскока, как развести тайминг нескольких элементов. Работает сам по себе, без
  каких-либо MCP-серверов.
- **`lottie-animator-skill`** знает строение Lottie JSON: порядок слоёв,
  transform-объект, безье-пресеты, parenting, покадровую анимацию.

Как грунт для валидности держите под рукой официальную спецификацию —
[Lottie Specs](https://lottie.github.io/lottie-spec/) (там же машинный JSON
Schema). С ней модель почти не выдумывает несуществующие поля.

## Основной путь: Claude Code пишет JSON

Дальше всё происходит в обычном диалоге. Опишите анимацию словами — чем
конкретнее, тем ближе первый результат:

```text title="Пример запроса"
Сделай Lottie-анимацию загрузки: три точки, пульсируют по очереди.
60fps, цикл 1.2с, мягкий ease-in-out. Сохрани в public/loading-dots.json.
```

Модель сгенерирует валидный `.json` и положит его в проект. Файлы анимаций держим
в `public/` — оттуда они отдаются от корня сайта (`/loading-dots.json`).

:::tip[В этом проекте так уже сделано]
Анимацию нейросети на старых страницах сгенерировал скрипт
`scripts/gen-neural-net-lottie.mjs` — он программно собирает `neural-net.json`.
Это тот же приём «код описывает Lottie», только зафиксированный в файле:
перегенерация командой `node scripts/gen-neural-net-lottie.mjs`.
:::

Не понравился результат — не переписывайте вручную, попросите доработать:
«сделай быстрее и добавь отскок на последней точке». Модель правит тот же JSON.

## Как просить хорошо

Качество почти целиком зависит от запроса. Несколько правил с большим эффектом:

- **Начинайте со своего SVG.** Рисовать фигуры с нуля модель умеет хуже, чем
  анимировать готовую графику. Дайте SVG-иконку — пусть отвечает за движение.
- **Именуйте слои осмысленно** (`left_dot`, `bg_circle`), а не `Shape 1`. Модель
  читает имена и точнее попадает в нужный элемент.
- **Дробите сложное на шаги.** Не «собери весь онбординг», а сцена за сценой:
  фигуры → ключевые кадры → доводка тайминга.
- **Конкретизируйте тайминг и характер.** Не «плавный фейд», а «fade-in 400ms,
  ease-out». Чем точнее про ощущение (резко, упруго, медленно) — тем меньше
  итераций.
- **Берите лучшую модель.** Тайминг и пространственные рассуждения заметно лучше
  на Opus/Sonnet, чем на мелких моделях.

Глубже про принципы движения — в [motion-design-skill](https://github.com/lottiefiles/motion-design-skill);
про структуру формата — в [Lottie Specs](https://lottie.github.io/lottie-spec/).

## Альтернативы

Основной путь покрывает большинство случаев. Иногда удобнее другое.

### kin3o — генерация из терминала

Локальный CLI: текст → валидный `.json` на диск, на вашей подписке Claude Code.
Сам валидирует результат и чинит типовые ошибки AI, умеет превью и экспорт в
видео. Аккаунт LottieFiles нужен только для публикации в их библиотеку — для
генерации не требуется.

```bash
npx @afromero/kin3o generate "pulsing loader, three dots" --provider claude-code
```

### Lottie Creator MCP — визуальный редактор

Если хочется не JSON, а живой редактор: Claude Code управляет веб-редактором
LottieFiles Creator через MCP, строит и правит анимацию слой за слоем, экспортирует
готовый файл.

```bash
claude mcp add lottiefiles-creator -- npx -y @lottiefiles/creator-mcp@latest
```

:::caution[Это уже не локально]
Creator MCP драйвит **облачный редактор** LottieFiles — нужен их аккаунт и
запущенный Creator в браузере. Локально остаётся только итоговый экспорт `.json`.
Берите этот путь, когда визуальный контроль важнее автономности.
:::

## Подводные камни

- **Размер файла.** Встроенные растровые картинки (base64) раздувают `.json` с
  десятков КБ до мегабайтов. Для вектора это не нужно — следите, чтобы модель не
  вставляла растр.
- **`prefers-reduced-motion`.** Уважайте системную настройку «уменьшить
  движение»: не запускайте автоплей для тех, кто его отключил.
