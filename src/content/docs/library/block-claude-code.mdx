Схематичный, настраиваемый мок интерфейса Claude Code. Нужен, чтобы показывать
«что где» в статьях-объяснениях, не прибегая к скриншотам.

## Превью

<ClaudeCodeFrame
	prompt="объясни, что делает этот модуль"
	response="Модуль маршрутизирует запросы между сервисами и кэширует ответы."
	model="Opus 4.8"
	context="42%"
/>

## Параметры

| Проп          | По умолчанию                      | Назначение              |
| ------------- | --------------------------------- | ----------------------- |
| `title`       | `claude`                          | подпись в шапке окна    |
| `prompt`      | `объясни, что делает этот модуль` | реплика пользователя    |
| `response`    | `Модуль отвечает за…`             | ответ ассистента        |
| `placeholder` | `спросите что-нибудь`             | текст в поле ввода      |
| `model`       | `Opus 4.8`                        | модель в статус-строке  |
| `cwd`         | `~/project`                       | рабочая папка в статусе |
| `context`     | `42%`                             | заполнение контекста    |

## Как вставить

Страница статьи должна быть `.mdx` (компоненты импортируются только в MDX).

```mdx
import ClaudeCodeFrame from '~/components/blocks/ClaudeCodeFrame.astro';

<ClaudeCodeFrame
	prompt="разбей задачу на шаги"
	response="1. Прочитать конфиг. 2. Сгенерировать клиент. 3. Прогнать тесты."
	context="18%"
/>
```
