## Правило

**Термины едины для всех статей.** Нельзя в одной статье писать «харнесс», в другой «агент», в третьей «хост». Всегда использовать термин из этого справочника.

## Термины

### Харнесс

**Определение:** среда исполнения агента — программа, которая управляет жизненным циклом: принимает задачу от пользователя, вызывает LLM-провайдера, исполняет инструменты, возвращает результат.

**Примеры харнессов:** Claude Code, Cursor, Pi Coding Agent.

**Не харнессы:** LangChain, LangGraph, CrewAI, Microsoft AutoGen — это фреймворки для построения агентов, а не среды исполнения.

**Запрещены:** агент (без уточнения), хост, среда, клиент.

**Контекст:** «Харнесс подключается к MCP-серверу», «Pi — это харнесс».

---

### Агент

**Определение:** LLM + системный промпт + доступные инструменты. Агент — это «мозг» внутри харнесса: получает задачу, рассуждает, выбирает инструменты, формирует ответ.

**Отличие от фреймворковых агентов:** агенты на LangChain/CrewAI — это программные конструкции, собранные разработчиком из цепочек и узлов графа. Агент в харнессе (Claude Code, Pi) — это сама LLM, действующая автономно в цикле «рассуждение → вызов инструмента → наблюдение → ответ». Харнесс даёт ей инструменты и исполняет вызовы, но не программирует логику.

**Запрещены:** харнесс (не путать — харнесс содержит агента, а не наоборот).

**Контекст:** «Агент выбрал инструмент jira.get_issue», «агент решает задачу по шагам».

---

### MCP-сервер

**Определение:** процесс, реализующий Model Context Protocol. Предоставляет агенту доступ к конкретной системе (Jira, файловая система, Chrome DevTools) через стандартизированный интерфейс инструментов.

**Запрещены:** MCP, сервер (без уточнения), плагин.

**Контекст:** «MCP-сервер для Jira», «харнесс подключается к MCP-серверу».

---

### Инструмент

**Определение:** функция, которую MCP-сервер предоставляет агенту. Агент вызывает инструмент через цепочку: харнесс → MCP-сервер → целевая система. В оригинальной документации MCP — _tool_.

**Запрещены:** тул, метод, функция, tool (англицизм).

**Контекст:** «MCP-сервер вернул список инструментов», «агент выбрал инструмент get_issue».

---

### LLM-провайдер

**Определение:** сервис, предоставляющий большую языковую модель по API (Anthropic, Google, OpenAI, self-hosted).

**Запрещены:** LLM (без уточнения), модель, облако, API.

**Контекст:** «Харнесс отправляет промпт LLM-провайдеру», «ответ от LLM-провайдера».

---

### Скилл

**Определение:** пакет инструкций и правил для агента, загружаемый харнессом. Расширяет возможности агента без изменения кода харнесса.

**Запрещены:** навык, расширение, плагин, аддон.

**Контекст:** «Скилл d2-diagrams помогает агенту рисовать диаграммы».

---

### Промпт

**Определение:** текстовый запрос от пользователя или харнесса к LLM. Может включать системные инструкции, контекст, список инструментов.

**Запрещены:** запрос (без уточнения), сообщение.

**Контекст:** «Харнесс прикрепляет инструменты к промпту».
